# Vault UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`. Autonomous loop: make sensible defaults, don't wait for the user, record ambiguity in the ledger, commit per task, DO NOT push.

**Goal:** A logged-in user (managed wallet) can create their agent policy + vault, deposit/withdraw USDC, and hit a PANIC button — all built client-side and executed through `POST /api/wallet/execute` against the mainnet vault package v2.

**Architecture:** Client builds `Transaction`s that call `0x9c49…::agent_policy` / `::vault` move functions, serializes with `tx.serialize()`, and POSTs `{txJson}` to the existing `/api/wallet/execute` (session-gated, allowlisted, server-signed). Reads (vault balance/positions/policy) come from a new server route querying the chain via `suiClient`. USDC on mainnet: confirm the coin type at build time (native USDC `0xdba34672e30cb065b1f93e3ab55318768fd6fef66c15942c9f7cb846e2f900e7::usdc::USDC` — verify against chain before hardcoding).

**Tech stack:** Next.js 16, `@mysten/sui@1.45.2`, the existing `lib/wallet/*` + `useManagedWallet` hook.

### Task 1: Transaction builders (`lib/wallet/vault-tx.ts`)
Pure client helpers returning serialized `Transaction`s (no signing). Read `talos/sources/*.move` for exact signatures first.
- `buildCreatePolicy({agent, budget, perTxCap, protocols, expiresAtMs})` → calls `agent_policy::create_policy_entry`.
- `buildCreateVault({policyId, allowedPositions})` → `vault::create_vault<USDC>`.
- `buildDeposit({vaultId, coinObjectId?, amount})` → `vault::deposit<USDC>` (split from a USDC coin the user owns).
- `buildOwnerWithdrawUsdc({vaultId, ownerCapId, amount})` → `vault::owner_withdraw_usdc<USDC>`, transfer the returned coin to the sender.
- `buildPanic({policyId, vaultId, ownerCapId})` → ONE PTB: `agent_policy::revoke` + `vault::owner_withdraw_usdc` (all idle), transfer to sender.
Each returns `tx.serialize()`-ready. Test: construct each, assert `Transaction.from(serialized).getData().commands` are all MoveCalls into package v2 (so `/api/wallet/execute` will accept them). Commit.

### Task 2: Vault-state read route (`app/api/wallet/vault/route.ts`)
GET, session-gated → returns the user's `{ policyId, vaultId, ownerCapId, idleUsdc, positions, revoked, remainingBudget }` by querying owned objects of the session address (find OwnerCap + shared Vault/AgentPolicy via events or owned objects) via `suiClient`. If none yet → `{ exists:false }`. Commit.

### Task 3: `useVault()` hook + create-on-first-visit
`lib/wallet/use-vault.ts`: fetches `/api/wallet/vault`; exposes `{ vault, loading, refresh }` and an `execute(txJson)` that POSTs to `/api/wallet/execute` then refreshes. If `!vault.exists`, expose `createVault()` that runs buildCreatePolicy → (wait) → buildCreateVault via execute. Commit.

### Task 4: Vault UI (`components/wallet/vault-panel.tsx`) + wire into `app/(app)/app/page.tsx`
A card showing idle USDC + deployed position + policy status; a Deposit input+button, a Withdraw input+button, and a red PANIC button (confirm dialog → buildPanic → execute). Disabled states while pending; show tx digest links to suiscan. Match the Talos dark/green aesthetic. Verify with a headless screenshot (logged-out shows sign-in; the vault panel renders behind a mocked/dev session if feasible, else verify build + component render in isolation). Commit.
